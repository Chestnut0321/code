# PPSS-Dual Objective Optimization
# 项目简介
本项目基于IBM CPLEX Optimization Studio，实现了**项目-供应商选择双目标优化模型**，核心目标为在约束条件下同时最大化项目利润与最小化供应商合作风险，适用于多周期、多资源、多供应商场景下的决策优化分析。
模型通过量化项目收益、采购成本、供应商风险等指标，结合资源需求、项目执行、供应商容量等约束，求解帕累托最优解，为企业资源配置与合作决策提供数据支撑。

# 核心特性
- 双目标优化：同步优化“利润最大化”与“风险最小化”，支持利润阈值约束下的风险最优求解。
- 多场景适配：支持自定义项目数量、供应商数量、周期数及资源类型，适配不同规模的决策场景。
- 批量测试能力：内置多组实例循环测试逻辑，自动输出每组测试的核心优化结果。
- 规范结果输出：结果文件自动归档至指定目录，清晰记录每组实例的利润、风险等关键指标。
- 内存安全管控：完善的动态内存分配与释放逻辑，避免大规模数据处理时的内存泄漏问题。

# 快速开始
1. 环境配置
安装CPLEX后，配置环境变量：
- Linux/macOS：在终端执行 export LD_LIBRARY_PATH=$LD_LIBRARY_PATH:/path/to/cplex/lib（替换为实际CPLEX安装路径）。
- Windows：在系统环境变量中添加CPLEX的 bin 和 lib 目录路径。
2. 数据准备
（1） 在项目根目录创建 data 文件夹。
（2） 按格式放入数据文件，命名规则：{项目数}-{供应商数}-({测试次数})data.txt（例：15-10-(1)data.txt 表示15个项目、10个供应商的第1组测试数据）。
（3） 数据文件需包含以下核心字段（顺序一致）：项目资源需求、供应商资源容量、采购单价、供应商风险值、项目周期、项目收益、周期政府补贴、周期预算、项目-供应商距离、项目间预算节省系数。
3. 编译运行

# 常见问题
1. 编译报错“无法找到-lcplex”
原因：CPLEX环境变量未配置正确，或编译时未指定CPLEX库路径。解决方案：
g++ src/main.cpp -o ppss -std=c++11 -I/path/to/cplex/include -L/path/to/cplex/lib -lcplex
2. 运行时提示“数据文件不存在”
检查 data 目录下的文件命名是否符合规则，且项目数、供应商数与代码中循环参数一致（默认循环15/20个项目、10个供应商）。
3. CPLEX求解无结果
可能原因：约束条件冲突，或时间限制过短。解决方案：调整约束逻辑，或增大 TIME_LIMIT 参数。
